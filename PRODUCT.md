## 一、产品概述

### 1.1 产品定位

**天问-AGI** 是一个天文研究领域的通用认知智能体系统，代号 Hermes-AGI，取自屈原《天问》，象征对宇宙万物的好奇与探索。

**核心范式：认知脑 + 执行肢**
- **天问-AGI**: 承担认知层，负责高层决策、任务规划、环境感知理解
- **具身智能系统**: 承担执行层，负责望远镜控制、实地观测执行

| 属性 | 描述 |
|-----|------|
| **产品名称** | 天问-AGI (Tianwen-AGI) |
| **英文名** | Hermes-AGI |
| **项目地址** | https://github.com/LL-LK/tianwen-agi |
| **版本** | v3.2.0 |
| **定位** | 天文研究的认知大脑 + 全自动天文观测站 |
| **愿景** | 探索智能本质，实现通用人工智能 |

### 1.2 核心价值主张

```
文献调研 ─────────────────────────────────────────────┐
    │                                                      │
    ▼                                                      │
制定观测计划 → 执行自动化观测 → 数据挖掘 → 产生新发现 ─────┘
                              │
                              ▼
                        指导下一轮调研
```

天问-AGI 的独特之处：**不是简单的问答系统，而是能够自主完成「文献调研→观测→数据挖掘→指导新观测」完整闭环的 AGI 原型系统**。

### 1.3 当前能力矩阵

| 能力领域 | 当前实现 | 成熟度 | 备注 |
|---------|---------|-------|------|
| **通用认知** | 三大引擎(认知/规划/执行) | 🟢 成熟 | 意图识别、任务分解、技能调度 |

### 4.2 P0 级风险详解（必须立即解决）

#### R1: 文献调研能力完全缺失 🔴

**问题描述**:
当前系统完全没有文献调研能力，无法完成"文献调研→观测→数据挖掘→指导观测"闭环的第一步。

**具体表现**:
- 没有 arxiv 论文搜索功能
- 没有研究现状分析（某天体目前知道什么、不知道什么）
- 没有研究热点识别
- 没有 Gap 分析（已知和未知的边界）

**影响**:
```
缺失文献调研 → 无法确定观测目标 → 盲目观测 → 数据价值低 → 无法产生新发现
```

**解决方案**:

| 模块 | 功能 | 优先级 |
|-----|------|-------|
| literature_researcher.py | arxiv 搜索、论文摘要、引用分析 | P0 |
| research_state_analyzer.py | 研究现状图谱、Gap 识别 | P0 |
| hypothesis_generator.py | 从文献中生成可检验假说 | P0 |

**实现建议**:
```python
# literature_researcher.py 核心接口
class LiteratureResearcher:
    def search_arxiv(self, query: str, max_results: int = 50) -> List[Paper]:
        """搜索 arxiv 论文"""
        
    def analyze_research_state(self, topic: str) -> ResearchState:
        """分析某领域研究现状"""
        # 返回: 已解决什么问题 / 正在研究什么 / 关键未解问题
        
    def identify_gaps(self, topic: str) -> List[ResearchGap]:
        """识别研究空白"""
        # 返回: 具体未研究的问题 + 为什么重要 + 如何研究
        
    def generate_hypotheses(self, gaps: List[ResearchGap]) -> List[Hypothesis]:
        """从研究空白生成可检验假说"""
```

#### R2: 数据挖掘能力薄弱 🔴

**问题描述**:
当前只有简单的统计分析和异常检测，缺乏从数据中发现新模式、新关系的能力。

**具体表现**:
- 只有基础的均值/方差分析
- 异常检测只用 IQR/Zscore，过于简单
- 没有特征工程能力
- 没有模式发现能力
- 没有假设检验机制

**影响**:
```
观测数据 → 简单统计 → 无法发现新模式 → 无法产生新发现 → 闭环断裂
```

**解决方案**:

| 模块 | 功能 | 优先级 |
|-----|------|-------|
| pattern_discovery.py | 从时序数据中发现周期性/趋势/异常模式 | P0 |
| feature_engineering.py | 自动特征提取和选择 | P0 |
| hypothesis_tester.py | 统计假设检验 | P0 |
| correlation_analyzer.py | 多变量相关性分析 | P1 |

---

#### R3: 无闭环反馈机制 🔴

**问题描述**:
系统无法记录、追踪、管理"发现→假说→验证"的全生命周期。

**具体表现**:
- 没有发现数据库
- 没有假说追踪系统
- 没有验证结果记录
- 没有"哪些假说被验证/推翻"的统计

**影响**:
```
新发现无法积累 → 知识无法增长 → 系统无法真正"学会"天文研究
```

**解决方案**:

| 模块 | 功能 | 优先级 |
|-----|------|-------|

### 4.3 P1 级风险详解

#### R4: 缺乏真实设备控制

**问题描述**:
当前只能采集公开 API 数据，无法控制真实天文设备（望远镜、相机等）。

**具体表现**:
- 无法连接真实望远镜（KStars/Ekos/INDI）
- 无法控制天文相机
- 无法进行实时星图匹配（plate solving）
- 无法进行圆顶/指向控制

**影响**:
- 只能"模拟观测"，无法进行真实天文研究
- 数据来源于他人，无法声称"原创发现"

**解决方案**:
```python
# equipment_controller.py
class EquipmentController:

        """拍摄天文图像"""
```

```python
@app.post("/api/chat")
```
