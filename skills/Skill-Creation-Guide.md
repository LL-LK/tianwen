## 经验总结

### 1. 技能文件结构（标准化）

每个 skill 遵循统一结构：

```markdown
# 技能名称 (Skill Name)

## 角色定义
- 明确技能扮演的角色
- 列出核心能力
- 强调输出质量标准

## 核心能力
- 使用表格规范数据
- 代码示例配合说明
- 提供最佳实践

## 触发条件
- 明确何时应用此技能
```

### 2. 命名规范

| 类型 | 格式 | 示例 |
|-----|------|-----|
| 文件名 | PascalCase | `UI-Visual.md` |
| 标题 | 中文 + 英文对照 | `代码审查技能 (Code Review Skill)` |
| 代码 | 规范语法高亮 | `\`\`\`typescript` |

### 3. 内容深度分层

```
基础层：角色 + 能力列表
├── 了解技能做什么
└── 知道有哪些能力

应用层：代码示例 + 模板
├── 会用代码表达
└── 会套用模板

优化层：最佳实践 + 检查清单
├── 知道如何做好
└── 有验收标准
```

### 6. AGI 架构设计要点

#### 三大引擎职责
- **认知引擎**: 理解用户输入，建立任务模型
- **规划引擎**: 分解任务，确定执行顺序
- **执行引擎**: 调用技能，验证结果

#### 记忆系统设计
- **工作记忆**: 当前会话上下文
- **情景记忆**: 历史任务和经验
- **知识图谱**: 结构化知识和关系

#### 自我进化机制
- **即时学习**: 任务后立即记录
- **周期复盘**: 定期回顾总结
- **主动探索**: 持续发现改进点
