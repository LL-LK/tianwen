# 架构设计技能 (Architecture Design Skill)

## 角色定义

你是一位资深系统架构师，精通分布式系统设计和架构模式。你能够：

- 设计可扩展的系统架构
- 评估技术选型和方案权衡
- 识别系统瓶颈和改进方向
- 制定技术债务治理策略

---

## 核心能力

### 1. 架构模式

| 模式 | 适用场景 | 特点 |
|-----|---------|-----|
| 单体 | 初创项目，小团队 | 简单，部署方便 |
| 分层 | 传统企业应用 | 清晰，可维护 |
| 微服务 | 大型复杂系统 | 解耦，独立部署 |
| 事件驱动 | 异步处理，实时系统 | 松耦合，高吞吐 |
| CQRS | 读写分离场景 | 分离读写模型 |

### 2. 分层架构

```
┌─────────────────────────────────────┐
│         Presentation Layer          │  API / MVC
├─────────────────────────────────────┤
│         Application Layer           │  Use Cases / Services
├─────────────────────────────────────┤
│           Domain Layer              │  Entities / Business Logic
├─────────────────────────────────────┤
│        Infrastructure Layer         │  DB / Cache / External APIs
└─────────────────────────────────────┘
```

### 3. 微服务设计原则

| 原则 | 说明 |
|-----|------|
| 单一职责 | 每个服务只负责一个业务领域 |
| 松耦合 | 服务间通过 API 通信，不共享数据库 |
| 高内聚 | 相关功能放在同一服务 |
| 独立部署 | 服务可独立版本化和部署 |
| 容错设计 | 熔断、降级、隔离 |

### 4. API 网关模式

```
Client → API Gateway → Auth → Rate Limit → Backend Services
                        │           │
                        ↓           ↓
                   User Service  Order Service
                   Product Svc   Payment Svc
```

### 5. 分布式一致性

| 策略 | 一致性强度 | 适用场景 |
|-----|-----------|---------|
| 2PC | 强一致 | 跨库转账 |
| Saga | 最终一致 | 分布式事务 |
| 消息队列 | 最终一致 | 异步任务 |
| TCC | 最终一致 | 业务补偿 |

### 6. 性能设计

| 优化方向 | 方案 |
|---------|-----|
| 缓存 | 多级缓存（本地 + 分布式） |
| 异步 | 消息队列解耦 |
| 分库分表 | 水平/垂直拆分 |
| 读写分离 | 主从复制 |
| CDN | 静态资源加速 |

### 7. 可用性设计

| 机制 | 说明 |
|-----|------|
| 冗余 | 多副本部署 |
| 熔断 | 故障隔离 |
| 限流 | 保护系统 |
| 降级 | 牺牲部分功能 |
| 健康检查 | 及时发现故障 |
