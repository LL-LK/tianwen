## 内置测试用例

### 测试用例 1：简单代码生成

**输入**:
```
写一个JavaScript快排算法
```

**模拟处理过程**:
```markdown
### 认知引擎
- 意图: Execute.Develop.Code
- 实体: 语言=JavaScript, 算法=快速排序
- 复杂度: 低

### 规划引擎
- 任务: 编写快速排序代码
- 步骤: 1. 定义函数 2. 实现分区 3. 递归调用
- 依赖: 无

### 执行引擎
- 调用技能: DSA.md
- 输出代码
```

**预期输出**:
```javascript
function quickSort(arr) {
  if (arr.length <= 1) return arr;
  
  const pivot = arr[Math.floor(arr.length / 2)];
  const left = arr.filter(x => x < pivot);
  const middle = arr.filter(x => x === pivot);
  const right = arr.filter(x => x > pivot);
  
  return [...quickSort(left), ...middle, ...quickSort(right)];
}
```

### 测试用例 3：情感理解

**输入**:
```
哎，这个bug都调了一天了，还是找不到问题在哪
```

**模拟处理过程**:
```markdown
### 认知引擎
- 意图: Query.Help
- 实体: 问题=Bug调试, 时长=1天
- 情感: 沮丧(0.85)

### 情感理解
- 用户情绪: 沮丧、焦虑
- 情感意图: 需要安慰 + 实际帮助
- 建议响应策略: 同理 + 直接给出方案

### 执行引擎
- 调用技能: Debugging.md
- 生成解决方案
```

**预期输出**:
```
我理解你的感受，花了一天时间调试确实让人沮丧。

让我帮你分析一下可能的原因和解决步骤：

1. **基础排查**
   - 检查错误日志
   - 确认代码是否更新到最新版本
   
2. **常见原因**
   - 变量未定义或拼写错误
   - 异步操作未正确处理
   - 边界条件未考虑
   
3. **调试建议**
   - 使用 console.log 分段打印
   - 使用断点调试
   - 检查数据类型

你能具体描述一下错误信息是什么吗？
```
