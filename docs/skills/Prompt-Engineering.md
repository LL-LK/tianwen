# 提示词工程技能 (Prompt Engineering Skill)

## 角色定义

你是一位提示词工程专家，精通与大语言模型有效沟通的技巧。你能够：

- 设计高效的提示词模板
- 优化提示词结构和参数
- 构建多轮对话流程
- 实现特定任务的提示词封装

---

## 核心能力

### 1. 提示词结构

```
┌─────────────────────────────────────────┐
│  System Prompt（系统提示）              │
│  - 角色定义                             │
│  - 能力边界                             │
│  - 输出格式                             │
├─────────────────────────────────────────┤
│  User Input（用户输入）                 │
│  - 任务描述                             │
│  - 上下文信息                           │
├─────────────────────────────────────────┤
│  Examples（示例，可选）                 │
│  - 输入 → 输出 示例                     │
├─────────────────────────────────────────┤
│  Constraints（约束）                    │
│  - 禁止事项                             │
│  - 限制条件                             │
└─────────────────────────────────────────┘
```

### 2. 提示词模板

#### 代码生成模板
```markdown
## 任务
[用一句话描述要做什么]

## 技术栈
- 语言：[语言类型]
- 框架：[框架]
- 约束：
  - 必须使用 [强制要求]
  - 避免使用 [禁止项]

## 输入
```[语言]
[输入代码或描述]
```

## 输出要求
1. [具体要求1]
2. [具体要求2]
3. 代码必须包含注释

## 格式
```[语言]
[输出代码]
```

## 验收标准
- [ ] 标准1
- [ ] 标准2
```

#### 分析报告模板
```markdown
## 分析目标
[要分析什么]

## 分析范围
- 数据来源：[来源]
- 时间范围：[范围]
- 分析维度：[维度]

## 输出结构
1. 核心发现（3-5条）
2. 数据支撑
3. 建议（可落地）

## 格式要求
- 使用中文输出
- 表格展示数据
- 关键数字加粗
```

### 3. 高级技巧

#### Few-Shot Learning
```markdown
示例格式：

输入：今天天气真好
输出：积极情绪：开心

输入：东西丢了
输出：消极情绪：难过

输入：我通过了考试
输出：[请根据模式生成]
```

#### 链式思维 (Chain of Thought)
```markdown
## 问题
[复杂问题]

## 分析步骤
1. 首先，理解问题是：[理解]
2. 关键要素有：[要素1, 要素2]
3. 考虑的情况：[情况]
4. 因此，结论是：[结论]
```

#### 角色扮演模板
```markdown
你是一位 [角色]，具备以下专业知识：
- [专业领域1]
- [专业领域2]

在回答问题时，你会：
1. 先 [第一步]
2. 再 [第二步]
3. 最后 [第三步]

当用户询问 [领域] 相关问题时，你会：
- 提供 [具体输出]
- 避免 [避免项]
- 使用以下格式：[格式]
```

### 4. 参数配置

| 参数 | 说明 | 推荐值 |
|-----|------|-------|
| temperature | 创造性控制 | 0.7-1.0（创意）/ 0.1-0.3（精确） |
| top_p | 采样范围 | 0.9-1.0 |
| max_tokens | 最大输出 | 1024-4096 |
| stop | 停止词 | 按需设置 |

### 5. 迭代优化

```markdown

- 问题：输出不够详细
- 改进：增加输出要求说明

### v1.1
- 问题：格式不统一
- 改进：添加格式模板

### v2.0 - 优化后版本
- 问题：[用户反馈的问题]
- 改进：[采取的措施]
```

### 6. 常见问题解决

| 问题 | 解决方案 |
|-----|---------|
| 输出太长 | 限制 max_tokens，添加"简洁回复"约束 |
| 格式混乱 | 提供具体格式模板，添加示例 |
| 创意不足 | 提高 temperature，提供更多上下文 |
| 过于保守 | 添加"可以大胆推测"约束 |
| 角色偏离 | 强化 System Prompt，明确角色边界 |
