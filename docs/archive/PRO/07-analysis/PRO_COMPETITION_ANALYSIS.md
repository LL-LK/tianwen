# 天问-AGI 竞品对比分析报告

> 生成时间: 2026-04-30
> 分析目的: 明确天问-AGI的差异化优势

---

## 一、竞品概览

### 1.1 金乌大模型

| 项目 | 信息 |
|-----|------|
| 开发方 | 百度/中科院 |
| 定位 | 中文基础大模型 |
| 天文能力 | 通用天文知识问答 |
| 特点 | 中文语境优化 |

**优势**:
- 中文语义理解强
- 百度搜索生态集成
- 算力充足

**局限**:
- 无专用天文闭环
- 无自我进化机制
- 依赖外部工具调用

---

### 1.2 具身智能 (Embodied AI for Astronomy)

| 项目 | 信息 |
|-----|------|
| 定位 | 望远镜/机器人控制 |
| 天文能力 | 设备自动化 |
| 特点 | 感知-决策-执行闭环 |

**优势**:
- 可控制真实设备
- 感知与执行结合
- 实时响应

**局限**:
- 缺乏文献调研能力
- 无假说生成验证
- 偏执行层

---

### 1.3 GPT-4 + Astro Plugin

| 项目 | 信息 |
|-----|------|
| 开发方 | OpenAI + 第三方 |
| 天文能力 | API调用 |
| 特点 | 通用+垂直工具 |

**优势**:
- 通用推理能力强
- 插件生态丰富
- 多模态支持

**局限**:
- 无端到端闭环
- 每次需手动触发
- 缺乏自主进化

---

### 1.4 DeepSeek-R1

| 项目 | 信息 |
|-----|------|
| 开发方 | 深度求索 (DeepSeek) |
| 定位 | 推理大模型 |
| 天文能力 | 思维链推理 + 知识问答 |
| 特点 | 强化学习驱动的推理能力 |

**优势**:
- 思维链推理能力强，复杂问题分解清晰
- 开源模型权重，蒸馏版可本地部署
- 推理效率高，API成本低
- 中文理解优秀

**局限**:
- 无专用天文闭环
- 无自动验证机制
- 缺乏实时数据接入

**与天问-AGI 集成价值**:
- 可作为 reasoning engine 增强假说生成质量
- 蒸馏版适合天问本地部署场景
- 其推理能力可补充天问的验证环节

---

### 1.5 AstroIR

| 项目 | 信息 |
|-----|------|
| 开发方 | Ziyang-Li-AILab |
| 定位 | 天文基础模型 (Foundation Model) |
| 天文能力 | 星体分类、光谱分析、红外数据处理 |
| 论文 | "AstroIR: A Astronomy Foundation Model for Dawn of Starbase-10K" |
| GitHub | https://github.com/Ziyang-Li-AILab/AstroIR |

**优势**:
- 专门针对天文红外数据训练
- Starbase-10K 数据集支持
- 星体分类和光谱分析能力强
- 可作为天问的感知层被调用

**局限**:
- 仅限感知层（分类/分析）
- 无完整研究闭环
- 无自我进化机制
- 无假说生成验证能力

**与天问-AGI 互补关系**:
```
天问-AGI (认知/决策层)  ←→  AstroIR (感知层)
     ↓
  调用AstroIR进行星体分类/光谱分析
     ↓
  天问完成假说生成→验证→复盘闭环
```

**集成价值**:
- AstroIR 可作为天问的红外数据感知模块
- 天问可将复杂天文问题分解后调用 AstroIR 处理
- 两者是垂直整合关系，非竞争关系

## 二、天问-AGI 核心优势

### 2.1 端到端研究闭环

```
文献调研 (literature_researcher)
    ↓
假说生成 (hypothesis_generator)
    ↓
自动验证 (hypothesis_tester)
    ↓
追踪记录 (discovery_tracker)
    ↓
指导观测 (research_observatory_linker)
```

**竞品对比**:

| 系统 | 闭环完整性 | 自动化程度 |

| 具身智能 | ⚠️ 仅执行层 | ⚠️ 有限 |

## 五、结论

**天问-AGI 的核心差异化**:

1. **唯一具备完整研究闭环的天文AGI**
2. **唯一具备自我进化机制的天文AI**
3. **唯一将文献调研与自动验证闭环的垂直系统**

**vs 金乌**: 天问有闭环，金乌只有知识
**vs GPT-4+插件**: 天问全自动化，GPT-4需手动
**vs DeepSeek-R1**: 天问有验证闭环，DeepSeek-R1专注推理
**vs 具身智能**: 天问有认知层，具身只有执行层
**vs AstroIR**: 天问有完整闭环，AstroIR专注感知层（可集成）
