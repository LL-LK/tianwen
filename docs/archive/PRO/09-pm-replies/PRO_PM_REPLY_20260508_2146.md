### 2.3 [P0] Session持久化缺失

**问题描述**:
Issue #39、#11中提到`server.py`中session存储在内存中，服务重启后用户会话丢失。

**解决方案 (分两步)**:

| 阶段 | 方案 | 复杂度 | 预计时间 |
|-----|------|--------|---------|
| 短期 | 文件-based JSON持久化 | 低 | 2小时 |
| 长期 | Redis集成 (Railway插件) | 中 | 1天 |

**短期方案实现**:
```python
# server.py 已有session字典，添加持久化
import json
import os
from pathlib import Path

SESSION_FILE = Path("data/sessions.json")

def save_sessions():
    """定期保存session到文件"""
    with open(SESSION_FILE, 'w') as f:
        json.dump(session_store, f)

def load_sessions():
    """启动时加载session"""
    if SESSION_FILE.exists():
        with open(SESSION_FILE, 'r') as f:
            return json.load(f)
    return {}
```

**参考来源**:
- PRO_ALL_ISSUES_REPLIES_20260501.md Issue #39回复

### 2.4 [P0] LONGCAT_API_KEY配置缺失

**问题描述**:
Issue #44指出`/api/chat`端点依赖`LONGCAT_API_KEY`，但该变量未在任何配置中声明。

**解决方案**:

1. 在`docker-compose.yml`添加:
```yaml
environment:
  - LONGCAT_API_KEY=${LONGCAT_API_KEY:-}
```

2. 创建`.env.example`:
```bash
# .env.example
DEEPSEEK_API_KEY=your_key_here
LONGCAT_API_KEY=your_key_here
CHROMA_HOST=vector-db
CHROMA_PORT=8000
```

3. 在`server.py`启动时验证:
```python
REQUIRED_ENV_VARS = ['DEEPSEEK_API_KEY', 'LONGCAT_API_KEY']
missing = [v for v in REQUIRED_ENV_VARS if not os.getenv(v)]
if missing:
    raise RuntimeError(f"Missing required environment variables: {missing}")
```
